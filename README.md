# WEIQ Scraper (天下秀商业数据爬虫)

基于 `Playwright` + `Pandas` 的 WEIQ 自动化数据提取脚本，专为应对复杂动态渲染、React/Vue 异步数据绑定以及商业级封控平台而生。

---

## 🆕 更新日志 (Changelog)

### 🚀 v2.0 史诗级更新 (最新)
**更新时间：2026-03-09**

本次更新对底层抓取引擎进行了脱胎换骨的重构，正式迈入商业级数据监控标准：
* **全量 15 项指标扩容**：单账号数据监控从 2 项暴增至 15 项。现已支持：粉丝数、直发CPM、阅读中位数（总/直发/转发）、互动中位数（总/直发/转发）、发布博文数、转发/评论/点赞中位数，以及极具价值的**最近 20 篇博文趋势（最低/最高/平均阅读量）**。
* **Viewport 懒加载击穿**：针对重负载图表按需加载的特性，注入“自动化平滑深度滚动算法”，强制唤醒 ECharts 与底部 AJAX 网络请求，告别漏抓。
* **父级区块逆向解析 (降维打击)**：在原有 TreeWalker 基础上，新增父区块结构化逆向解析策略。无视 DOM 多层嵌套、CSS 混淆及图标干扰，精准提取核心数值。
* **死链与空页诊断**：自动识别博主下架、UID 变更或未被平台收录的空白页，并在终端进行清晰播报，防止程序死锁。
* **旧表隔离机制**：启动时智能校验 `weiq_results.xlsx` 表头结构，发现不兼容自动加上时间戳备份并生成全新净表，彻底杜绝数据错位污染。

## 🌟 核心功能特性

* **精准定位**： 抛弃脆弱的传统 XPath 定位，采用高级 `JS TreeWalker` + 父区块解析，基于人类视觉流寻找目标数字，无视目标网站各类反爬虫、多层 div 嵌套及样式混淆。
* **凭证持久化 (免受限登录)**： 引入独立运行的大厅扫码/验证手机号的前置交互机制，捕获最纯净的 `state.json` 登录态，以后无论跑多少次均全部免登录。
* **熔断与冷却防封机制**：
  * 内置模拟真人阅读的动态延迟休眠（3-6秒随机倒排）。
  * 内置全局风控熔断：每执行全链路跳转访问 50 次，程序立刻强制休眠 3 分钟并进行可视化倒计时播报。
* **终端 GUI 化进展**： 实时打印 `[当前数/总计任务数]` 的任务处理进度，直观的倒计时读取条及动态状态反馈。
* **实时安全落盘**： 利用强容错设计与增量写入（`openpyxl` overlay），防止单个账号读取超时、空缺、封锁导致整个程序崩溃而丢尽已爬的数据。

## 🛠️ 环境依赖与使用说明

本项目严格采用当今最高速的 `uv` 工具链进行管理与执行。

### 1. 源文件准备
请在此目录下放置一份名为 `accounts.xlsx` 的 Excel 数据源文件。 表头的必填列名为：
* **账号ID**：用于在终端展示进度标记。
* **uid**：用于自动拼接生成指向 `https://weiq.com/client/product/weibo/detail?account_uid=xxxxxxxx` 的底层链接。

### 2. 环境初始化
在此目录下开启终端（Terminal），依次初始化依赖与浏览器驱动：
```bash
uv add pandas openpyxl playwright
uv run playwright install chromium
```

### 3. 一键运行爬虫
```bash
uv run python scraper.py
```
> **首次运行注意**：首次运行时，需要根据弹出的浏览器提示手工输入手机号或扫码登录，**成功进入大厅后切勿关闭浏览器**，回到终端敲击回车即可全自动接管。 
> *(如果遇到 `Permission denied` 错误，代表有人或 WPS 正在同时查看生成出来的 `weiq_results.xlsx` 表格，将其关闭后重新运行即可恢复)*

## 🍎 在 MacBook 上运行指南

**1. 准备基础环境**
首先，确保您的 MacBook 已经安装了 Python 和 uv（如果还没装 uv，在终端执行 `curl -LsSf https://astral.sh/uv/install.sh | sh` 即可一键安装）。

**2. 把代码拉下来**
打开 MacBook 的终端（Terminal），把项目克隆下来：
```bash
git clone [https://github.com/youfei0719/weiq-scraper.git](https://github.com/youfei0719/weiq-scraper.git)
cd weiq-scraper
```

**3. 安装依赖和驱动**
初始化虚拟环境并下载 Mac 版的 Chromium 浏览器内核引擎：
```bash
uv sync
uv run playwright install chromium
```

**4. 放数据，开始跑！**
把您的 `accounts.xlsx` 表格拖进这个克隆下来的文件夹里。最后，执行熟悉的命令起跑：
```bash
uv run python scraper.py
```

💡 **两个跨平台的小提示（Mac 版特权）：**
* **不用担心路径问题**：代码中使用了 `os.path.exists()` 等内置抽象库，未硬编码 `C:\` 等 Windows 专属路径，因此在 Mac 上文件读取、生成以及 `state.json` 的存取完全正常。
* **文件锁会温柔很多**：Mac 系统的文件锁机制不像 Windows 那么“暴政”。在某些版本的 Mac Excel 中，即便您开着结果表，程序偶尔也能静默把新数据写进去（但保险起见，建议运行的时候还是关掉 Excel 结果表格）。
